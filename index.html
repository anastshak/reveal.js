MongoDB

What is it

MongoDB – the most popular NoSQL database (the term 'NoSQL' means 'non-relational'). Also it is an open-source document-oriented database.

Concept

MongoDB works on concept of database, collection and document.

  • Database is a physical container for collections.
  • Collection is a group of MongoDB documents.
  • A document is a set of key-value pairs.

RDBMS vs MongoDB

  • RDBMS has a typical schema design, unlike MongoDB is document-oriented.
  • Complex transactions are not supported in MongoDB.
  • MongoDB allows a highly flexible and scalable document structure.
  • MongoDB is faster.

Stored data

MongoDB isn't based on the table-like relational database structure, so it's stores data in flexible, JSON-like documents (this format of storage is called BSON), meaning fields can vary from document to document and data structure can be changed over time.

A simple MongoDB document structure:


							{
								'title': 'Presentation',
								'by': 'Anastasia Shakura',
								'url': 'https://...',
								'type': 'MongoDB-NoSQL'
							}

Below is a sample document in MongoDB:


					{
						"_id" : ObjectId("52832eb59f36fe144eeea8dc"),
						"baseprice" : 8.99,
						"category" : "toys",
						"colors" : [ "red", "green", "cosmic purple"],
						"name" : "Cosmic Yo-yo",
						"promotions" : [ 
							{ "coupon" : "XY678", "saleprice" : 7.99 },
							{ "coupon" : "AB8888", "saleprice" : 7.49 }
						]
					}

About ID:

ObjectId is a 12-byte BSON type, constructed using:

  • a 4-byte value representing the seconds since the Unix epoch,
  • a 3-byte machine identifier,
  • a 2-byte process id, and
  • a 3-byte counter, starting with a random value.
